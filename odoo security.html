Odoo Security System (4 Layer)

Category  
↓  
Group  
↓  
Access Control (ir.model.access.csv)  
↓  
Record Rule

1️⃣ Security Category

Category ব্যবহার করা হয় group গুলো organize করার জন্য।

Example:

<record  id="module_category_hospital"  model="ir.module.category">  
  <field  name="name">Hospital</field>  
</record>

Result:

Settings → Users → Groups

এখানে Hospital নামে category দেখাবে।

2️⃣ User Group

Group ব্যবহার করা হয় user role define করার জন্য।

Example:

Receptionist  
Doctor  
Manager

Code:

<record  id="group_hospital_receptionist"  model="res.groups">  
  <field  name="name">Receptionist</field>  
  <field  name="category_id"  ref="school.module_category_hospital"/>  
</record>

Result:

Hospital  
 ├── Receptionist  
 ├── Doctor

User কে group assign করা হয়।

3️⃣ Access Control (ir.model.access.csv)

id,name,model_id:id,group_id:id,perm_read,perm_write,perm_create,perm_unlink
access_patient,access_patient,school.model_hospital_management,base.group_user,1,1,1,0

4️⃣ Record Rule

Access control বলে:

কি করা যাবে

Record rule বলে:

কোন record করা যাবে

Example:

Doctor শুধু নিজের patient দেখতে পারবে।

<record  id="doctor_patient_rule"  model="ir.rule">  
  <field  name="name">Doctor Own Patient</field>  
  <field  name="model_id"  ref="school.model_hospital_management"/>  
  <field  name="domain_force">[('doctor_id','=',user.id)]</field>  
</record>

Meaning:

Doctor শুধু নিজের patient দেখতে পারবে।

আমরা যে কোডটা দেখছি:

<field  name="model_id"  ref="school.model_hospital_management"/>  
<field  name="domain_force">[('doctor_id','=',user.id)]</field>

এটা Odoo Record Rule এর domain।

1️⃣ doctor_id কোথা থেকে আসে?

doctor_id হলো model-এর একটি field।

Example: তোমার model যদি এমন হয়:

from  odoo  import  models, fields  
  
class  HospitalManagement(models.Model):  
  _name  =  'hospital.management'  
  
  name  =  fields.Char(string="Patient")  
  
  doctor_id  =  fields.Many2one(  
  'res.users',  
  string="Doctor"  
 )

এখানে:

doctor_id = fields.Many2one('res.users')

মানে:

Patient → Doctor (User)

তাই record rule-এ doctor_id ব্যবহার করা হচ্ছে।

2️⃣ user.id কোথা থেকে আসে?

user হলো current logged-in user।

Odoo automatically domain-এ user variable দেয়।

Example:

user.id

মানে:

current logged-in user id

9️⃣ Important Concept

Record rule always model field ব্যবহার করে।

domain_force → model fields

তাই:

doctor_id

model field থেকে এসেছে।

ami ki akhana doctor ka user hisaba doraci or usergroup.

1️⃣ User vs User Group

  • user: যেই মানুষটি Odoo তে লগইন করছে। (res.users)

  • user group: permission বা role assign করার জন্য। উদাহরণ: Doctor, Receptionist, Manager। (res.groups)

2️⃣ তুমি কি করেছো?

<record  id="doctor_patient_rule"  model="ir.rule">  
  <field  name="name">Doctor Own Patient</field>  
  <field  name="model_id"  ref="school.model_hospital_management"/>  
  <field  name="domain_force">[('doctor_id','=',user.id)]</field>  
</record>

  • এটা বলছে: সব doctor শুধু নিজের patient দেখবে।

  • কিন্তু যে user এই rule এ apply হবে সেটা এখন সব user। অর্থাৎ, receptionist, manager, সবাই এ rule এ পড়বে।

3️⃣ কিভাবে শুধু Doctor কে apply করা যায়?

তোমাকে rule-এ groups field add করতে হবে। যেমন:

<record  id="doctor_patient_rule"  model="ir.rule">  
  <field  name="name">Doctor Own Patient</field>  
  <field  name="model_id"  ref="school.model_hospital_management"/>  
  <field  name="domain_force">[('doctor_id','=',user.id)]</field>  
  <field  name="groups"  eval="[(4, ref('school.group_hospital_doctors'))]"/>  
</record>

  • groups field use করলে rule apply হবে শুধু ওই group-এর users এর উপর।

  • school.group_hospital_doctors হলো সেই group যেটা তুমি doctor এর জন্য তৈরি করেছো।
